## 1. 專案簡介
本專案使用 STM32 NUCLEO-L452RE 作為主控制器，結合 MPU6050 姿態感測器、TB6612FNG 馬達驅動器、雙直流減速馬達與 TCRT5000 循跡感測器，製作一台雙輪自走平衡車。

目前整理的程式版本為**「Balance Only（僅平衡控制）」**，循跡功能與 TCRT5000 接線仍保留於系統規劃中，但程式控制已暫時關閉，以便優先將車體平衡調適穩定。

## 2. 系統功能
- **姿態估測**：讀取 MPU6050 加速度與角速度，並使用互補濾波估測車體傾角。
- **平衡控制**：透過角度 PID 控制輸出 PWM，使雙輪車維持平衡。
- **馬達驅動**：TB6612FNG 控制左右馬達正反轉與轉速。
- **狀態監控**：UART 輸出角度、PWM、MPU 讀取狀態，方便透過 Tera Term 等工具即時除錯。
- **擴充性**：硬體上已保留兩顆 TCRT5000 感測器，未來可輕易加入黑線循跡功能。

## 3. 硬體周邊清單
| 類別 | 元件 | 用途 |
|---|---|---|
| 主控制板 | STM32 NUCLEO-L452RE | 負責讀取感測器、計算控制量、輸出 PWM 與方向訊號。 |
| 姿態感測器 | MPU6050 | 讀取加速度與角速度，用於估測平衡角度。 |
| 馬達驅動 | TB6612FNG | 控制左右兩顆直流馬達。 |
| 循跡感測器 | TCRT5000 x2 | 未來用於黑線循跡，目前硬體保留。 |
| 執行器 | 雙直流減速馬達 | 平衡車左右輪動力來源。 |
| 電源 | 12V 電池 / DC-DC 降壓模組 | 提供馬達電源與控制板邏輯電源。 |

## 4. 關鍵接線表
> **⚠️ 注意：STM32、感測器、馬達驅動、電源模組的 GND 必須全部共地。**

### MPU6050 (姿態感測器)
| MPU6050 | NUCLEO-L452RE | 說明 |
|---|---|---|
| VCC | 3.3V | 感測器電源 |
| GND | GND | 共地 |
| SCL | PB8 / I2C1_SCL | I2C 時脈 |
| SDA | PB9 / I2C1_SDA | I2C 資料 |

### TB6612FNG (馬達驅動)
| TB6612FNG | NUCLEO-L452RE | 用途 |
|---|---|---|
| PWMA, PWMB | PB14, PB15 | 左右馬達 PWM 輸出 |
| AIN1, AIN2 | PA4, PA5 | 左馬達方向控制 |
| BIN1, BIN2 | PA6, PA7 | 右馬達方向控制 |
| STBY | PB0 | 驅動器啟用 |
| VM | 12V | 馬達輸入電源 |
| VCC | 3.3V 或 5V | 邏輯電源 |

## 5. 快速開始與使用方法
1. **硬體組裝**：依據上述接線表完成硬體與模組的連接，並確認電源共地。
2. **匯入專案**：將本專案匯入 STM32CubeIDE，確認 `main.c` 已正確載入。
3. **編譯與燒錄**：編譯專案並使用 ST-Link 將程式燒錄至 NUCLEO-L452RE。
4. **系統啟動與校正**：
   - 開啟 Tera Term 等 UART 監控軟體（Baud rate: 115200, 8N1）。
   - 系統重新上電或按 Reset，等待終端機畫面出現 `Gyro calibrating...`（此時請確保車體靜止勿晃動）。
5. **啟動平衡控制**：
   - 校正完成後，**用手扶住車體**並保持在你想要的平衡角度。
   - 按下 NUCLEO 板上的**藍色 USER Button**。
   - 看到終端機印出 `BALANCE ONLY START` 後，系統即開始輸出 PWM 進行平衡控制。

## 6. 調參建議 (PID)
主要的 PID 控制參數位於 `main.c` 的全域變數區段：
- `Kp = 26.0f`：角度比例控制。反應太慢可加大；太容易暴衝可降低。
- `Kd = 3.2f`：角速度阻尼。左右來回震盪可加大；反應太鈍可降低。
- `Ki = 0.0f`：積分控制。在基礎平衡未穩定前建議保持為 0。
- `MOTOR_MIN_PWM = 85`：補償馬達死區。若馬達有聲音但推不動可提高數值。
